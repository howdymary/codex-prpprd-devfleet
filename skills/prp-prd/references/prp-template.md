# PRP Template

```md
# <Feature Name> PRP

## Objective
- One short paragraph connecting this plan back to the PRD.

## Technical Shape
- High-level implementation approach
- Why this approach fits the repo

## Touch Points
- `path/to/file.ext`: why it changes
- `path/to/other.ext`: why it changes

## Data / State Changes
- Schema, model, cache, config, env, or migration implications

## API / Interface Changes
- Backend endpoints, function signatures, CLI flags, UI contracts, or event shapes

## Execution Sequence
1. Step 1
2. Step 2
3. Step 3

## Parallelizable Work
- Packet A
- Packet B

## Testing Plan
- Unit or small-scope verification
- Integration or behavior verification
- Manual smoke check

## Observability / Debugging
- Logs, metrics, dashboards, or debug affordances to inspect

## Rollout / Safety
- Feature flag, staged rollout, migration ordering, rollback path, or fallback behavior

## Risks / Tradeoffs
- Tradeoff
- Why it is acceptable or how to mitigate it

## Definition Of Done
- [ ] Code path implemented
- [ ] Tests updated or added
- [ ] Docs/config updated if needed
- [ ] Reviewer handoff prepared
```
